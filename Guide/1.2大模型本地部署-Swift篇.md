# 一  先了解Swift
`modelscope/ms-swift` 是 **阿里ModelScope 社区**推出的一个 **大模型与多模态大模型微调、推理、评测、量化与部署框架**，支持 **600+ 文本大模型** 和 **400+ 多模态大模型** 的全流程训练与部署

## swift核心功能

- **支持模型类型**
    - 文本大模型：Qwen3/3.5、InternLM3、GLM4.5、Mistral、DeepSeek-R1、Llama4 等
    - 多模态大模型：Qwen3-VL、Qwen3-Omni、Llava、InternVL3.5、MiniCPM-V-4、Ovis2.5、GLM4.5-V 等
- **训练方式**
    - 全参数训练 & 轻量化微调（LoRA、QLoRA、DoRA、LoRA+、LongLoRA、Adapter 等）
    - 支持 **Megatron 并行**（TP、PP、CP、EP、SP、VPP 等）和 **分布式训练**（DDP、FSDP、DeepSpeed ZeRO2/3）
- **强化学习 (RLHF)**
    - 内置 GRPO、DAPO、GSPO、SAPO、CISPO、RLOO、Reinforce++ 等算法
    - 支持 DPO、KTO、RM、CPO、SimPO、ORPO 等偏好学习
- **推理与部署**
    - 支持 Transformers、vLLM、SGLang、LMDeploy 推理加速
    - 提供 OpenAI API 接口
- **量化支持**
    - GPTQ、AWQ、BNB、FP8、AQLM、HQQ 等多种量化方案
- **多模态能力**
    - 文本、图像、视频、音频混合训练
    - 支持独立控制 vit/aligner/llm 模块
- **内置数据集**
    - 150+ 预训练、微调、人类对齐、多模态任务数据集
    - 支持自定义数据集一键训练

## 本地模型调用
```python
# 从 swift 导入
from swift import InferRequest, RequestConfig, TransformersEngine
# 1. 创建实例模型，使用具体引擎类：TransformersEngine
engine = TransformersEngine(
    model="./models/Qwen2.5-1.5B-Instruct"
)
# 2. 构建请求
request = InferRequest(
    messages=[{"role": "user", "content": "用50个字以内介绍一个植物"}],
)

# 3. 构建配置参数
request_config = RequestConfig(temperature=0.7, max_tokens=512)
'''
print(request_config)
这是一堆超参数，可以加上print看看自己的model是什么样的
RequestConfig(max_tokens=512, temperature=0.7, top_k=None, top_p=None, repetition_penalty=None,num_beams=1, stop=[], seed=None, stream=False, logprobs=False, top_logprobs=None,n=1, best_of=None, presence_penalty=0.0, frequency_penalty=0.0, length_penalty=1.0,return_details=False, structured_outputs_regex=None)
'''
# 4. 执行推理
response = engine.infer([request], request_config=request_config)
print("🤖 模型回答:", response[0].choices[0].message.content)

```


## 总之SWIFT只要这几步
1. 加载你的本地大模型 TransformersEngine model写上文件位置，不用去做复杂的分词en decode 截取回复token  
2. 直接发送推理请求InferRequest 在请求里把我们的上下文messages传进去
3. RequestConfig方法 传递实参来控制超参数，通过赋值来后续给engine.infer，当然你也可以把一堆超参数直接写在engine.infer里，就像这样
4. ```
   response = engine.infer([request], temperature=0.7, max_tokens=512)
   ```
5. 使用engine.infer 方法你不再需要手动截取 token，框架返回的直接是干净的回答这一步融合了发送messages 和调整超参数和返回回复文本
6. print回复



## api-key调用

就是和本地的差不多，只是说InferEngine配置有一点不一样
```python
from swift import InferEngine

engine = InferEngine(
	backend="openai",# 关键：切换为api模式
	model_name="deepseek-chat", # 模型名称,不同模型有不同要求，不能随便填，ds就是这个
	api_key="sk-xxxxxxxxxxxxxxxx", #你自己的
	api_base="https://api.deepseek.com"  #ds就是这个网址
)

response = engine.infer( 
	["你好，介绍一下自己"], #省略了messages = 什么什么
	temperature=0.7, # 直接写超参数 
	max_tokens=512 ) 
	
print(response)

```
