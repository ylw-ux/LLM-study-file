### 1.2.1项目文件结构与命名避坑指南

对于纯小白来说，“文件存在哪”和“怎么命名”往往是引发莫名其妙报错的重灾区。第三方库（比如 `openai`）通过 `pip install` 安装后，会自动存放在你 Python 环境的 `site-packages` 目录下，**你完全不需要手动去下载或移动它们的源文件**。

你在自己的电脑上新建一个项目文件夹，建议采用以下的结构划分：
``` 
my_llM_project/                 # 你的项目主文件夹
│
├── models/                     # 用来存放下载到本地的大模型权重文件（体积很大）
│   └── Qwen2.5-1.5B-Instruct/  # 具体的模型文件夹
│
├── api_keys.env                # 配置文件，用来存放你的 API 密钥（千万别传到网上）
│
├── call_deepseek_api.py        # 脚本1：测试调用 DeepSeek API
├── run_local_qwen.py           # 脚本2：使用 Transformers 跑本地模型
├── deploy_swift.py             # 脚本3：使用 SWIFT 框架部署
│
└── requirements.txt            # 记录你这个项目安装了哪些包及版本
```

**极其重要的命名规则（小白必看）：**

- **绝对不要**把你的 Python 脚本命名为你要引入的库的名字！例如，千万不要把你的文件命名为 `openai.py`、`requests.py` 或 `transformers.py`。如果你这么做了，Python 会优先读取你的文件，导致“找不到该模块里的方法”这种让小白抓狂的异常。
    
- 路径中尽量避免中文和特殊空格。
### 1.2.2创建并激活“虚拟环境”

大模型需要的包（比如 `openai`）更新极快，为了不弄脏你电脑上原有的 Python 环境，我们要建一个“独立小房间”。

在vscode里打开的终端里，依次输入以下命令并回车：

**1. 创建虚拟环境：**
```Bash
python -m venv llm_env
```

(这行命令的意思是：调用 Python 内置的 venv 模块，在当前文件夹下创建一个名叫 `llm_env` 的虚拟环境。执行后稍微等几秒，左侧文件列表会多出一个 `llm_env` 文件夹。)
**2. 激活虚拟环境：**

- **如果你是 Windows 系统，输入：**
``` Bash
    .\llm_env\Scripts\activate
```
- **如果你是 Mac/Linux 系统，输入：**    
``` Bash
    source llm_env/bin/activate
```

成功标志：
**激活后，你的终端命令行最前面会出现一个绿色的 `(llm_env)` 字样。这意味着你接下来的所有操作，都只发生在这个小房间里了。

### 异步版本：应对高并发
以上都是同步代码，单个请求没问题。但如果你要同时发起几十个请求（比如批量处理文档、并行调用多个模型比较结果），同步方式就不够用了——每个请求要几秒，串行跑完需要几分钟。这时候用 asyncio + AsyncOpenAI：



```python
# hello_async.py
import asyncio
import os
from openai import AsyncOpenAI
from dotenv import load_dotenv

load_dotenv()

client = AsyncOpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com/v1",
)

async def ask(question: str) -> str:
    resp = await client.chat.completions.create(
        model="deepseek-chat",
        messages=[{"role": "user", "content": question}],
    )
    return resp.choices[0].message.content

async def main():
    questions = [
        "用一句话解释什么是 GIL",
        "用一句话解释什么是事件循环",
        "用一句话解释什么是协程",
    ]
    results = await asyncio.gather(*[ask(q) for q in questions])
    for q, r in zip(questions, results):
        print(f"Q: {q}\nA: {r}\n")

asyncio.run(main())

```
三个请求并行发出，总耗时约等于最慢那一个，而不是三个相加。实际工程中处理百万级文档都靠这套模式。如果你还没学过 async/await，现在只要知道它存在，等我们做 RAG 批量向量化的时候再深入。
